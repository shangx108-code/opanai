# 规划-第一轮

## 1. 项目主标识与当前定位

- 统一项目主标识：`field-free-jj-active-channel-family-ncomms-20260513-1510`
- 固定长期目录：`/workspace/memory/projects/field-free-jj-active-channel-family-ncomms-20260513-1510/`
- 当前目标期刊：`Nature Communications`
- 当前稿件主线：把现有稿件从“严谨但偏负结果的数值审计”升级为一篇 `matched-control benchmark / evidence-standard` 论文；如果后续找到真正通过 `scalar + profile + finite-size + CAR/EC` 全 ladder 的 active-channel family，则把它作为 benchmark paper 里的 strongest positive outcome，而不是当前就把全文押成 device discovery 叙事。

## 2. 当前阶段

- 当前处于：`NC 定位重锁 + 项目统筹初始化`
- 已确认的基础：
  - 主文与补充已经形成一条可投稿但尚不够 NC 稳健的主线。
  - wide-link 结果已经足够强，能承担“topology-only does not imply finite-device evidence”的 baseline。
  - active-channel 结果已经足够说明：`scalar release`、`active-profile dominance`、`finite-size/end-environment stability`、`CAR-selective transport` 不是同一个东西。
  - 当前上传的修改意见已经把文章应当如何改写、补证据、重建主图和重排方法学说得很清楚。

## 3. 当前唯一主瓶颈

- 唯一主瓶颈：`证据链还没有闭环到 NC 审稿人无法直接一刀否掉的程度`
- 更具体地说，这个瓶颈包含两个相互关联但应视作同一个首要问题的子风险：
  - 风险 A：审稿人会质疑 `matched-control ladder` 是否严格到“任何真实拓扑系统都过不了”。
  - 风险 B：即便 benchmark 框架成立，如果最终仍没有一个 active-channel family 通过全 ladder，文章容易被理解成“一篇高质量但仍偏负向的诊断报告”，而不是 field standard paper。
- 因此，本轮之后的所有推进都必须优先服务于一句核心判断：
  - `这个 ladder 既不是人为过严，也不是仅对单一模型成立；同时它确实能区分 topology-only positives、scalar-only positives、profile-only artifacts、finite-size artifacts 和 CAR-nonselective transport。`

## 4. 总体投稿策略

- 最高层目标不是“先把现有稿子润色到更好看”，而是：
  - 先把 paper center 从 `false positives` 改成 `matched-control evidence benchmark`
  - 再把 benchmark 的可释放性、稳健性、跨 profile/texture 普适性和 transport 判别力补全
  - 最后才做全稿重写、主图重建和投稿包整理
- 当前 safest NC 叙事：
  - `Topology is a selector, not finite-device evidence.`
  - `Matched trivial controls, profile-resolved localization, finite-size/end-environment audit and CAR/EC decomposition define the necessary evidence ladder for field-free Josephson topology.`
- 当前 best-case NC 叙事：
  - 在上述 benchmark paper 骨架不变的前提下，再加入一个真正通过全 ladder 的 active-channel family，证明这套 protocol 既能拒绝假阳性，也能释放真正值得讨论的 candidate family。

## 6. 后续工作包总表

### WP0. 文献与主线重锁

- 目标：
  - 把标题、摘要、引言 opening、Results spine、Discussion take-home 全部重锁到 `benchmark / evidence standard` 叙事。
- 输入：
  - 当前主文、补充、上传的修改意见。
- 输出：
  - 新标题候选
  - 新摘要框架
  - 新引言五段式骨架
  - 新 Results 八小节目录
- 完成标准：
  - 文稿不再以 `false positives` 作为标题中心
  - 主文每一节都服务于 evidence ladder，而不是分散展示结果
- 优先级：`最高`

### WP1. known-positive benchmark 校准

- 目标：
  - 证明 `matched-control ladder` 不是“永远拒绝”的 protocol。
- 推荐执行路线：
  - 先做 `Kitaev chain benchmark` 作为最短闭环；
  - 若结果干净，再考虑更接近主模型的 `standard Zeeman planar JJ benchmark`。
- 必需输出：
  - candidate vs matched trivial controls
  - scalar `B` score
  - finite-size scaling
  - 最小 transport proxy（可选）
- 完成标准：
  - known-positive topological benchmark 能稳定通过 scalar matched-control gate
  - trivial controls 不能通过
  - 可以在 rebuttal 中明确写出：`the protocol does not reject finite-device topology by construction`
- 依赖：
  - 仅依赖现有 matched-control protocol 定义，适合立即启动
- 优先级：`最高`

### WP2. control-pool sensitivity 全重算

- 目标：
  - 证明结论不是 `strongest-control outlier` 人工造成。
- 统一对比标准：
  - `max control`
  - `95th percentile control`
  - `top-5 mean control`
- 需要重算的数据面：
  - wide-link release fraction
  - active-channel scalar windows
  - scalar-positive / profile-negative 象限计数
  - transport `|G|` / `CAR` / `near-zero CAR` pass fraction
  - strongest false-positive representative rows
- 输出：
  - 一张 control sensitivity matrix
  - 一张 Extended Data figure
  - 一个 reviewer-safe 结论段
- 完成标准：
  - wide-link null 至少在多种 control 定义下仍基本成立
  - scalar/profile decoupling 不依赖单个 outlier
  - transport magnitude vs CAR decoupling 仍保留
- 优先级：`最高`

### WP3. active-channel family 全 ladder 搜索

- 目标：
  - 在 benchmark 主线不变的前提下，尽最大概率找到一个真正通过 `scalar + profile + finite-size + CAR/EC` 全 ladder 的 active-channel family。
- 搜索原则：
  - family 不是单点；至少需要多长度或多参数窗口上的持续通过
  - 不允许只靠 scalar positive 就当作 device-positive
  - 不允许只靠 `|G|` positive 掩盖 CAR-selective 失败
- 重点搜寻区域：
  - 当前最强的 width-4 families
  - 已有 scalar-positive/profile-positive 候选
  - transport all-pass 或 near-all-pass 周边窗口
- 输出：
  - family evidence table
  - candidate ranking
  - full-ladder pass/fail ledger
- 完成标准：
  - 若找到全 ladder family，则作为主文高级结果
  - 若未找到，也必须形成清楚的 family-level failure map，而不是零散负结果
- 优先级：`高`

### WP4. finite-size / end-environment 机制审计

- 目标：
  - 彻底审查 `L = 176` 以及相关长度窗口到底是 resonance 还是 sensitivity。
- 需要输出的量：
  - lowest `6-10` BdG eigenvalues vs `L`
  - splitting vs `L`
  - minigap proxy vs `L`
  - total end weight vs `L`
  - active-channel weight vs `L`
  - scalar `B` vs `L`
  - active-profile margin vs `L`
  - normal extension / smoothing 对比
- 必答问题：
  - 是否存在 level crossing / avoided crossing
  - strongest control 是否在特定长度附近产生 end-weighted trivial state
  - scalar release 是否只是 total-end accumulation，而不是 active-channel localization
  - boundary quantization 是否被 normal extension 改写
- 完成标准：
  - 若证据足够，则保留 `resonance`
  - 若证据不足，则统一改写为 `finite-size/end-environment sensitivity`
- 优先级：`高`

### WP5. cross-texture / cross-profile robustness

- 目标：
  - 避免文章被审稿人压缩成“只对一种 d-wave-like compensated texture 和一种 active-channel geometry 有效”。
- 最低变体集：
  - rotated compensated texture
  - alternative harmonic
  - shifted / asymmetric / smooth-barrier active channel
- 输出：
  - cross-platform evidence matrix
  - Extended Data robustness figure
- 完成标准：
  - 文章能安全地说 benchmark 框架不只绑定一个 profile/texture 实现
- 优先级：`中高`

### WP6. random disorder ensemble

- 目标：
  - 把当前 deterministic perturbation 扩成统计 ensemble。
- 最小代表点：
  - wide-link near-positive but unreleased
  - active-channel scalar-positive/profile-negative
  - active-channel scalar-positive/profile-positive 候选
  - transport `|G|`-positive/CAR-negative
- 每个点最小输出：
  - `30` 个 onsite disorder seeds
  - `30` 个 interface disorder seeds
  - matched candidate/control seeds
  - scalar `B` distribution
  - active-profile margin distribution
  - transport pass fraction with confidence interval
- 完成标准：
  - 能给出一句统计结论，而不是只展示几条 deterministic robustness trace
- 优先级：`中高`

### WP7. transport blind validation 与 reverse audit

- 目标：
  - 把 transport 从“有意思但易被质疑”提升为 benchmark ladder 的第四级。
- 必补项：
  - freeze training / held-out split before scoring
  - contact selection rule pseudocode
  - integration energy window
  - 至少一个额外 lead coupling
  - 对唯一 held-out all-pass row 做 reverse audit
- 输出：
  - all-settings funnel
  - held-out funnel
  - `|G| ratio` vs `CAR ratio` scatter
  - strongest-control energy envelope
  - all-pass row reverse-audit sheet
- 完成标准：
  - transport 章节能够清楚支持一句话：
    - `magnitude release is not CAR-selective release`
- 优先级：`高`
