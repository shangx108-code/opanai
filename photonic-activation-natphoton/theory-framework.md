# Theory Framework

## Working purpose
Translate the paper's main intuition into a compact and falsifiable mathematical framework that can support both numerical benchmarks and manuscript writing.

## Core statement
For a photonic neural-network layer, the difficult resource is not the unitary linear map but the realization of a nonlinear activation under quantum noise, finite sampling, and device imperfections. We therefore define nonlinear activation as a constrained physical channel-design problem.

## Unified activation model

Let the pre-activation optical input be represented by an encoded scalar variable `x` carried by an optical state `rho_x`. A physical activation channel is written as

`A_theta : rho_x -> y`

with effective output

`y = E[g_theta(m, u) | rho_x, eta, ell, xi, N_s]`

where

- `m` is the measurement outcome, which may be absent for intrinsic-material activations
- `u` is a control variable generated by feed-forward or feedback logic
- `theta` is the trainable or programmable parameter set
- `eta` is detector efficiency
- `ell` is optical loss
- `xi` collects nuisance noise such as dark counts, phase noise, thermal noise, electronic noise, and calibration drift
- `N_s` is the effective shot budget or number of repeated samples used to estimate the activation output

This covers:

- electronic activation: detection plus electronic nonlinear transform plus modulation
- Kerr-like activation: intrinsic intensity-dependent phase or amplitude response
- saturable activation: material-response nonlinearity
- photon-counting activation: threshold or count-conditioned output
- homodyne activation: quadrature-conditioned output
- AQMA: weak tap, measurement, Bayesian or threshold logic, and conditional optical modulation

## Target activation approximation

Let `sigma(x)` denote a desired target activation function over the operating domain `x in [x_min, x_max]`.

Define the weighted approximation loss

`E_act(theta) = integral_{x_min}^{x_max} w(x) |A_theta(x) - sigma(x)|^2 dx`

where `w(x)` is the task-dependent input density or a benchmark weighting.

This quantity measures functional faithfulness only. It does not yet say whether a network task benefits from the approximation.

## Metric 1: Activation quantum cost

Define the activation quantum cost as the minimum average optical resource needed to realize a target activation within tolerance `epsilon`:

`C_AQC(sigma, epsilon) = min over theta { n_bar(theta) : E_act(theta) <= epsilon }`

where `n_bar` is the average photon number consumed per activation event at the chosen operating point.

### Interpretation
- This is the paper's central quantity.
- It asks how expensive a nonlinear response is in the few-photon regime.
- It is meaningful only after the operating assumptions are stated: encoding convention, input range, detector model, and whether repeated sampling is allowed.

## Metric 2: Discrimination cost

Because activation is useful only if it helps separate nearby inputs, define a discrimination score for two nearby encoded inputs `x_1` and `x_2`:

`D(x_1, x_2; theta) = |mu_1 - mu_2| / sqrt(var_1 + var_2)`

with `mu_i = E[y | x_i]` and `var_i = Var[y | x_i]`.

Then define the minimum resource required to exceed a desired discrimination threshold `D_0`:

`C_disc(D_0) = min over theta { n_bar(theta) : min over (x_1, x_2) in S D(x_1, x_2; theta) >= D_0 }`

where `S` is a benchmark set of difficult input pairs around the activation's turning region.

### Interpretation
- `C_AQC` evaluates function imitation.
- `C_disc` evaluates whether the activation is physically informative under noise.
- The paper should avoid claiming usefulness from `C_AQC` alone.

## Metric 3: Task-level energy-accuracy cost

For a fixed network architecture and training protocol, define

`C_task(A_0) = min over theta, architecture, and operating point { E_inf : Accuracy >= A_0 }`

where `E_inf` is the optical energy per inference, including linear propagation, taps, detectors, and conditional modulation overhead where relevant.

### Interpretation
- This is the device-facing metric for Figure 5 style results.
- It determines when a physically expensive activation still pays off at task level.
- It must be compared against strong baselines, not only against an ablated no-activation network.

## AQMA working model

For the first benchmark cycle, use the threshold-photon-counting form as the reference AQMA instance:

`A_AQMA(x) = a_0 + a_1 * P(n >= k | x, eta, ell, xi)`

This gives a sigmoid-like family controlled by threshold `k`, coupling coefficients `(a_0, a_1)`, tap ratio, and detector model.

The second AQMA instance should use homodyne-conditioned Bayesian estimation:

`A_AQMA-h(x) = E[sigma(z) | X_theta, model]`

This is useful when arguing that measurement-induced activation can be tuned continuously rather than only thresholded.

## Assumptions that must remain explicit

1. Encoding rule from logical pre-activation `x` to optical state `rho_x`
2. Whether `n_bar` includes local-oscillator power for homodyne baselines
3. Whether repeated shots count as time overhead, energy overhead, or both
4. Whether conditional modulation latency is ignored, estimated, or explicitly modeled
5. Whether detector dark counts and finite dead time are included
6. Whether comparison against electronic activation includes detector and modulator energy, not only optical energy

## What would count as a publishable theoretical result

The framework becomes manuscript-worthy if the project can show at least one of the following:

1. A regime boundary: below a certain photon budget, no candidate physical activation can approximate a sharp target nonlinearity without unacceptable discrimination loss.
2. A superiority window: AQMA or a related measurement-induced activation dominates alternative photonic activations under realistic `eta`, `ell`, and `N_s`.
3. A no-free-lunch result: some seemingly attractive activations achieve low approximation error but fail on discrimination cost or task-level cost.

## Immediate numerical consequences

- Single-neuron benchmarks should report `C_AQC`, `C_disc`, and response curves together.
- Small-network benchmarks should report `C_task` and show where the ranking of activations changes.
- Hardware maps should not report accuracy alone; they should display the frontier between useful, marginal, and not-worth-it regions.
