# 角色任务表：计算成像2

## 统筹者
- 当前任务：维护项目阶段判断、主瓶颈、唯一最高优先级和跨模块依赖关系
- 输入：上传项目草案、期刊门槛、各角色最新真实产出
- 输出：每轮状态更新、优先级调整、里程碑判定
- 完成标准：任何时刻都能明确当前阶段、当前主瓶颈、本轮最高优先级、交付物和下一轮动作
- 依赖：全部角色的真实完成状态
- 优先级：最高

## 理论人员
- 当前任务：把第一轮真实结果推进为 OIG / PIE / TIG 可检查推导链
- 输入：统一 forward model、噪声模型、先验设定、任务变量定义、第一轮真实结果
- 输出：
  - 统一数学符号表
  - 线性高斯互信息推导
  - OIG、PIE 第一版严格定义文稿
  - TIG 第一版定义
  - reconstruction-optimal 与 task-optimal 差异的命题草案
- 完成标准：
  - 每个关键结论都有连续推导，而不是概念口号
  - 清楚区分已证明结果、近似结果和待验证命题
- 依赖：第二轮 task-weighted 目标与仿真参数固定
- 优先级：P1

## 代码与数值计算人员
- 当前任务：把已跑通的最小线性高斯容量平台扩展到 task-weighted 第二轮
- 输入：任务定义、编码基线、噪声模型、评价指标、第一轮结果
- 输出：
  - 线性 forward model 与最小脚本入口
  - identity / random / Hadamard / binary mask 第一轮基线结果
  - task-weighted / prior-matched 第二轮扩展结果
  - mutual information、Bayes risk、PIE、TIG 对照表
  - 运行日志与失败案例记录
- 完成标准：
  - 第一轮脚本已真实运行
  - 输入输出接口明确
  - 第二轮结果能够检验排序是否翻转
- 依赖：理论人员给出 task 定义与评价形式
- 优先级：P1

## 数据分析人员
- 当前任务：建立第一轮结果判读表，并准备第二轮排序翻转检查
- 输入：baseline 输出、参数表、互信息结果、风险曲线、后续 TIG 结果
- 输出：
  - 指标解释模板
  - OIG / PIE / Bayes risk 一致性检查表
  - 第一版异常结果归因记录
  - 第二轮排序翻转判读表
- 完成标准：
  - 能解释不同编码为何提升或损害容量
  - 能指出需要补充的对照与敏感性分析
- 依赖：代码与数值计算结果
- 优先级：P2

## 画图人员
- 当前任务：把首版 SVG 总览图推进到投稿级 Figure 2 字段规范
- 输入：图 1-6 草案、统一变量命名、第一轮结果字段、第二轮任务字段
- 输出：
  - 图表风格规范
  - Figure 1 概念图要素清单
  - Figure 2 数据映射表与面板草图
  - 现有 SVG 总览图的面板拆分方案
- 完成标准：
  - 图表模板能直接承接后续真实结果
  - 不把任何示意图误当正式结果
- 依赖：统筹者确定图表结构
- 优先级：P3

## 论文撰写人员
- 当前任务：建立稿件骨架，不提前写结论
- 输入：论文定位、期刊标准、图表规划、理论框架、第一轮真实结果
- 输出：
  - 标题与摘要骨架
  - Introduction 论证链骨架
  - Results / Methods / Supplementary 章节骨架
- 完成标准：
  - 写作结构与 Nature Communications 匹配
  - 所有主张都明确标记其所需证据
- 依赖：统筹者与理论人员
- 优先级：P3

## 监督人员
- 当前任务：检查是否把局部 baseline 结果包装成一般性结论
- 输入：全部角色产出
- 输出：
  - 阶段性质检意见
  - 禁止提前宣称完成的条目
  - 下一轮风险提醒
- 完成标准：
  - 所有未闭合证据链都被明确标记
  - 所有不满足 First Principle 的内容都不计入完成项
- 依赖：全部角色
- 优先级：最高

## 严格审稿人
- 当前任务：暂不启动正式五审稿人循环，只维护预审入口条件
- 输入：阶段性结果、图表、推导、文稿
- 输出：是否具备进入严格审稿循环的判定
- 完成标准：
  - 只有当至少形成一版可审稿稿件与核心图表后，才进入正式五审稿人评估
- 依赖：理论、结果、图表、写作均形成可审材料
- 优先级：后置

## 当前依赖主链
1. 统筹者锁定第二轮 task-weighted 目标
2. 理论人员固定 TIG 的第一版定义
3. 代码与数值计算人员扩展第二轮结果
4. 数据分析人员检查是否出现排序翻转
5. 画图人员据此固定 Figure 2 和 Figure 4 最小接口
6. 撰写人员再写第一版 Results 与 Methods 骨架

## 本轮唯一最高优先级任务拆解
- 任务名称：线性高斯 task-weighted 第二轮
- 负责人：统筹者 + 理论人员 + 代码与数值计算人员
- 预期输出：
  - 第二轮参数表
  - task-weighted 扩展脚本入口
  - 第一版 TIG 对照结果表
  - 与理论定义对齐的排序翻转观察记录
- 完成标准：
  - 不是“想做第二轮”，而是“第二轮脚本真实跑通并留下结果”
