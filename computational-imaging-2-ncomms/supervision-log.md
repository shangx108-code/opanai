# 监督日志：计算成像2

## 监督轮次 0
- 日期：2026-04-25
- 节点性质：项目初始化监督

## 当前版本总体评价
项目定位强，具备跨平台统一理论的潜力，但当前仍是高质量 proposal，不是可投稿研究成果。

## 与 Nature Communications 标准的差距
1. 缺少真实运行结果，尚无证据证明容量指标和关键结论成立。
2. 缺少闭合理论推导，当前主要是框架性表述。
3. 缺少正式图表，Figure 1-6 仅为规划。
4. 参考文献远未补足，相关工作链条不完整。
5. 尚无正文和补充材料草稿可供严格审稿。

## 已达标部分
- 研究问题重要且具有跨平台普适性。
- 期刊定位基本合理。
- 主线从“设计更好的编码器”提升到“给出统一容量理论和设计原则”，理论高度足够。

## 未达标部分
- 理论严谨性
- 代码与数值结果
- 数据完整性
- 图表成熟度
- 写作完成度
- 审稿准备度

## 核心质量风险
最大风险是项目长期停留在“理论想法很好，但没有第一条真实证据链”的状态，导致后续任何写作都只能是包装，而不是研究推进。

## 证据链缺口
- OIG、TIG、PIE、RCR 尚无严格推导闭环
- 线性高斯最小 benchmark 未跑通
- 尚未证明 reconstruction-optimal、task-optimal、robust-optimal 之间的差异确实出现
- 尚未建立真实 Figure 2 数据链

## 必须纠正项
1. 立即停止把路线图和图表规划当结果表述。
2. 先做最小线性高斯平台，不同时铺开四个平台。
3. 在出现第一批真实结果前，不进入正式写作和正式审稿循环。
4. 将参考文献扩充到 30 篇以上，并完成主题分类。

## 优先级调整建议
保持“先线性高斯容量闭环，后扩展到 Fisher / task / robust 三条验证线”的路线，不提前展开大规模图表美化和长篇写作。

## 是否允许进入下一阶段
暂不允许进入“结果生成完成”或“成稿”阶段。

允许进入的下一步仅为：
- 基础平台搭建
- 第一批真实结果生成
- 理论推导细化
- 文献系统扩充

## 下一轮监督重点
1. 线性高斯最小脚本是否真实跑通
2. OIG 与 PIE 是否开始从口号变为可计算对象
3. Figure 2 是否开始出现真实数据字段
4. 文献扩充是否开始有系统推进

## 监督轮次 1
- 日期：2026-04-26
- 节点性质：第一轮真实计算后监督

## 当前版本总体评价
项目已经从“只有路线图”进入“有第一条真实运行结果”的状态，这是实质性前进。但目前这条证据链仍只够支撑 baseline 事实，不够支撑论文核心命题。

## 与 Nature Communications 标准的差距
1. 目前只有第一轮 reconstruction-oriented 线性基线结果。
2. 还没有 task capacity 与 robust capacity 的真实计算。
3. 还没有详细推导把 OIG、PIE、TIG、RCR 连成统一理论。
4. 还没有投稿级 Figure 2，多面板图和补充分析仍缺失。
5. 文献仍停留在 12 篇种子状态。

## 已达标部分
- 已有真实脚本、真实结果表、真实总结文件。
- 已有一张基于真实结果生成的 SVG 总览图。
- 已能用真实结果而不是口号来判断 direct baseline、random mixing、Hadamard 和 binary mask 的第一轮排序。

## 未达标部分
- 理论闭环
- 任务容量结果
- 鲁棒容量结果
- 投稿级图表成熟度
- 文献链完整性
- 可审稿文稿

## 核心质量风险
新的主要风险不是“完全没有结果”，而是“过早把第一轮 baseline 结果写成一般性结论”。当前结果受先验与 throughput 假设影响很大，必须继续做第二轮和后续平台验证。

## 证据链缺口
- 缺少 TIG 的真实定义与计算
- 缺少 prior-matched 或 learned proxy 设计
- 缺少排序翻转或 trade-off 的强证据
- 缺少对 throughput surrogate 合理性的推导说明

## 必须纠正项
1. 不得把“identity 在当前设定下最好”表述成普适规律。
2. 立即进入第二轮 task-weighted 计算，检验排序是否翻转。
3. 开始补写 OIG 与 PIE 的详细推导，并说明 surrogate 假设。
4. 启动文献系统扩充，不能一直停在 12 篇。

## 优先级调整建议
将下一轮主任务改为“task-weighted 第二轮线性实验 + TIG 定义”，暂不分散到 PSF 或鲁棒平台。

## 是否允许进入下一阶段
允许保持在“结果生成已启动”阶段，但不允许宣称核心理论已被验证。

## 下一轮监督重点
1. 第二轮是否真实出现 reconstruction-optimal 与 task-optimal 排序差异
2. TIG 是否完成第一版可计算定义
3. Figure 2 / Figure 4 的数据接口是否开始闭合
4. 文献数是否开始向 30+ 推进

## 监督轮次 2
- 日期：2026-04-26
- 节点性质：第二轮执行规范锁定后监督

## 当前版本总体评价
项目没有新增第二轮真实数据，但完成了一个必要的前置收敛：第二轮 task-weighted 线性实验终于被压缩成唯一明确的执行规范。这一步降低了“目标正确但无法落地”的风险，属于真实推进，但不能包装成结果完成。

## 与 Nature Communications 标准的差距
1. 仍然没有第二轮真实 task capacity 数据。
2. 仍然没有 OIG、PIE、TIG、RCR 的连续详细推导。
3. Figure 2 仍未形成正式多面板投稿图。
4. 文献仍未扩充到 30 篇以上。
5. 正文与补充材料仍未形成可审稿版本。

## 已达标部分
- 第二轮任务变量、指标、参数网格、输出字段和排序翻转判定标准已经统一。
- 当前阶段、主瓶颈与角色依赖关系已经与该规范对齐。
- 已明确声明本轮完成的是“执行规范锁定”，不是“第二轮真实结果”。

## 未达标部分
- 第二轮真实代码运行
- 第二轮正式数据表
- TIG 详细推导
- 投稿级图表
- 文献链和文稿链

## 核心质量风险
若下一轮仍不按锁定规范真实运行代码，这个项目会从“没有明确任务”转变为“有明确任务但迟迟不执行”，依然会卡在证据链起点。

## 证据链缺口
- `T_lowpass` 与 `T_sparseROI` 只是被定义为最小任务族，尚未用真实数据比较排序
- TIG 第一版仅完成计算口径锁定，尚未完成连续推导
- 尚未确认 throughput penalty 改变时任务排序是否稳定
- Figure 2 仍缺少真实多面板字段填充

## 必须纠正项
1. 下一轮必须直接运行第二轮 task-weighted 线性实验，不能再次停留在规范层。
2. 所有文档中必须继续明确区分“规范已锁定”和“结果已生成”。
3. 在第二轮出数前，不得声称 task-optimal 与 reconstruction-optimal 已经分离。
4. TIG 若未完成详细推导，只能称为第一版计算定义。

## 优先级调整建议
维持唯一最高优先级不变：下一轮直接执行第二轮真实计算。文献扩充和写作骨架可以并行准备，但不得挤占主任务。

## 是否允许进入下一阶段
允许保持在“理论强化与结果生成中的第二轮任务容量规范锁定”阶段，不允许进入“第二轮结果已完成”或“成稿”阶段。

## 下一轮监督重点
1. 第二轮脚本是否真实运行完成
2. 是否产生 task Bayes risk 与 TIG 正式结果表
3. 是否出现至少一种稳定的排序翻转或清晰 trade-off
